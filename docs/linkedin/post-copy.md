# LinkedIn post kit for huggingfaceR

## Recommended post

If you work in R and need to analyze text, audio, images, or messy documents, this update is for you.

I just merged a major update to `huggingfaceR`, an R package for using Hugging Face models without needing Python by default.

It is built for people who already live in R: data scientists, researchers, analysts, economists, journalists, social scientists, and teams working with text, documents, audio, or images.

You can use it to:

- classify survey responses, reviews, tickets, or open-ended text
- create embeddings for search, clustering, matching, and topic discovery
- extract structured fields from messy notes, interviews, PDFs, or reports
- transcribe audio clips and analyze the transcript in R
- classify images and detect objects
- generate images from prompts
- build LLM workflows with chat, streaming, and tool calling
- download files from the Hugging Face Hub and inspect model providers

I also rebuilt the README and pkgdown site with live rendered examples, not just code snippets. The docs now show real output from public starter assets: MLK audio for transcription and MS COCO images for vision examples.

The goal is to make modern ML models feel natural from R: tidy inputs, tidy outputs, and examples you can run.

What would you try first in R: survey text, interviews, embeddings, or image tagging?
